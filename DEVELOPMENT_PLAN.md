## 4단계 — 완성도 및 발표 준비 (05/19 ~ 05/말, 약 1주)

**목표: 시연 가능한 완성품**

- [ ] 크래시 및 예외 상황 전수 점검
- [ ] 화면 전환 애니메이션 다듬기
- [ ] 실 기기 테스트 (다양한 해상도)
- [ ] 앱 아이콘, 스플래시 최종 완성
- [ ] Pretendard 폰트 적용
- [ ] 발표용 시나리오 동선 확인

### 발표 시연 동선 (예시)
```
앱 실행 → 스플래시
→ 로그인 화면
→ 회원가입 (이름/학년/이메일/관심사)
→ 온보딩
→ 홈 (오늘 학사일정 + 공지 미리보기)
→ AI 채팅 (질문 입력 → 답변 수신)
→ 공지 탭 (목록 → 상세)
→ 정보 탭 (프로필 확인)
```

## 리스크 및 대응

| 항목 | 리스크 | 대응 방안 |
|------|--------|-----------|
| AI 채팅 API | 서버 준비 지연 | 더미 응답으로 UI 먼저 완성 |
| 카카오 로그인 | 키 등록 / 심사 소요 시간 | 3단계 후반으로 미루기 |
| 학사일정 데이터 | 공식 API 없을 수 있음 | 하드코딩 or 크롤링 검토 |
| 서버 미완성 | API 전체 지연 | 2단계 완성 상태로 발표 가능하도록 유지 |

---

## 핵심 원칙

> **2단계까지는 API 없이 완전히 동작하는 앱을 만들고, 3단계에서 실제 데이터로 교체한다.**
>
> 서버 일정이 밀려도 앱 자체 완성도는 유지되어야 하며, 더미 데이터 → 실 데이터 교체가 Repository 레이어만 수정으로 가능하도록 설계한다.
